# Git Basics

## What is Git
Git is a version control system. It helps track changes in files over time.

## Important terms

### Repository
A repository is a project folder tracked by Git.

### Commit
A commit is a saved snapshot of changes.

### Branch
A branch is a separate line of work.

### Merge
A merge combines changes from one branch into another branch.

### Conflict
A conflict happens when Git cannot automatically decide which change to keep.

### Revert
Revert creates a new commit that undoes an old commit safely.

### Push
Push sends local commits to GitHub.

### Pull
Pull gets changes from GitHub to local machine.

## Basic workflow
1. Create or edit files
2. Check changes with `git status`
3. Stage changes with `git add`
4. Save changes with `git commit`
5. Send changes with `git push`
